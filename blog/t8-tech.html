SQL Injection - Ooooh Scary...

GOOOOOOOOOOOD Morning! Your favorite Town Crier Timmy Here!

Let's start things off with this illustration.

Kids, do not try this at home, but I am about to crudely illustrate one method to hack a computer and steal information from it. We residents here at Gabriel-man island are proud and upstanding citizens aren't we?

SQL injection is one of the ways you can steal data from an organization, particularly from their SQL database. What this exploits is how SQL will intepret certain pieces of data as commands for the fell means of the attacker. For example, an attacker may "inject" these string literal escape characters for field in a table when what it does is escape out of it and instead executes a command that will send everything in that table to the attacker!

One way for an organization to mitigate this is to make sure they filter their inputs for these strings of code. Of course, then one will think, wouldn't there then be malicious strings of code that can bypass that filter as well? Sure, and that's one of the reasons why web security is a huge problem in networks outside of Gabriel-Man Island. Intrusion detection mechanisms like firewalls and patching do little to prevent these attacks. Instead, to prevent and detect the attacks, you should perform thorough audits, make sure you use best coding practice to sanitize your web applications, and regularly perform a web security audit after each change and addition to your web components.
