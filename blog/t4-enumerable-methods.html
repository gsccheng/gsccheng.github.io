How Great is the Enumerable "Group_by"? Great!

The #group_by enumerable is great! Enumerables are mixins. They are what they sound like, modules that are mixed into multiple classes (so you don't have to keep redefining it for each class) so that they can also use these methods. In order for these classes to actually use these mixed in enumerable methods, it needs to first define a #each method so that it is able to pick itself (#self) apart. These picked off portions are then fed to the enumerable methods definitions to use. If you have the object itself with the enumerable and no block after it, the resulting object is considered to have an enumerator class.

I have yet to use group_by, but I'm sure the opportunity will come soon. We all know how beneficial hashes can be. Can you imagine a function that can parse through an array or a group of elements and divide them depending on the result into a hash? I can see it making life so much easier.

The syntax goes like this:

group_by { |obj| block } â†’ a_hash

For example, say you want to divide an array of numbers into separate arrays depending on whether they're multiples of 3. Furthermore, you want to make the distinction of what their remainders are. So, calling

[1,2,3,4,5,6].group_by { |i| i % 3}

will give

#=> {0=>[3, 6], 1=>[1, 4], 2=>[2, 5]}

You can see that the multiples of 3 are conveniently stored as values in the 0 key. Meanwhile, the other remainder results are stored in their own keys as well. Is that not great?!
