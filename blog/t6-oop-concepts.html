What is the difference between Classes and Modules?

Source: stackoverflow.com



We have worked somewhat extensively on Ruby classes these last few weeks. We learned that classes are basically a template for creating objects. It includes the methods and attributes that an object is formed in. Classes themselves can also inherit these methods and attributes from other classes that we can super or parent classes. Although a class can only directly inherit from one superclass at a time, that superclass may also inherit from another class.



Meanwhile a few weeks ago we were introduced to the concept of an enumerable. We began using them, finding that they are incredibly useful in simplifying code and reducing the need to explicitly state loops. We researched into what the enumerable is where we found out that an enumerable is a module. We may have mostly left it at that, perhaps only going as far as stating that the module is a mix-in for classes so they can use it. But what does a module actually look like?



The table below basically breaks it all down. The module is a collection of methods, constants, and classes. Although you can use the methods inside as instance methods for the class it is in, it is discouraged not to since it may make reading through and debugging the class very difficult to pin down where the method is defined from. So, because classes can only inherit one from one class a time, modules sort of act as this fine tuning for you to inject certain methods into different classes that you want to share similar behaviors. You include a module into a class by writing the following (replace ModuleName with the actual name) inside the top level of the class


include ModuleName
